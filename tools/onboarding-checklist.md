# New Project Onboarding Checklist

Use this checklist when starting a new client project using this repository as a template. Complete each step in order to ensure a consistent, professional project setup.

---

## 🗂️ Repository Setup

- [ ] Use this repo as a template (click **"Use this template"** on GitHub)
- [ ] Name the new repo using the convention: `client-[clientname]-[project]` (e.g., `client-acme-rebrand`)
- [ ] Set repository visibility (private for client work)
- [ ] Add the client name and project description to the new repo's About section
- [ ] Invite relevant team members as collaborators with appropriate permissions
- [ ] Enable Issues and Projects in repository Settings

---

## 📁 Project Files

- [ ] Copy and fill out [project-brief.md](../templates/project-brief.md) in the new repo
- [ ] Copy and customize [proposal-template.md](../templates/proposal-template.md) if needed
- [ ] Update the README of the new repo with client/project-specific details
- [ ] Create a `/assets` folder for client-provided logos, images, and documents
- [ ] Create a `/deliverables` folder for final client-facing documents

---

## 🎨 Branding & Design

- [ ] Obtain client brand assets (logo files, brand guide, color palette)
- [ ] Store brand assets in `/assets/branding/`
- [ ] Review [brand-guidelines.md](../branding/brand-guidelines.md) for HarrisConsulting standards
- [ ] Review [color-palette.md](../branding/color-palette.md) and note any client palette overrides
- [ ] Confirm typography decisions for deliverables

---

## 📋 Project Management

- [ ] Create a GitHub Project board (Issues → Projects → New Project)
- [ ] Set up columns: Backlog, In Progress, In Review, Done
- [ ] Create opening issues for each major milestone from the project brief
- [ ] Assign milestone dates to issues
- [ ] Set up Labels: `deliverable`, `research`, `client-feedback`, `blocked`
- [ ] Schedule kickoff meeting with client

---

## 📞 Client Onboarding

- [ ] Send welcome email with project overview and timeline
- [ ] Share access to any shared tools (Google Drive folder, Notion workspace, etc.)
- [ ] Confirm communication channels (email, Slack, Teams, etc.)
- [ ] Set up recurring check-in cadence (weekly/biweekly sync)
- [ ] Collect signed contract / Statement of Work
- [ ] Collect initial payment per payment terms

---

## 🔐 Admin & Legal

- [ ] Confirm NDA / confidentiality agreement is signed and on file
- [ ] Confirm data handling requirements (any sensitive data involved?)
- [ ] Add project to internal time tracking system
- [ ] Create invoice draft(s) per the payment schedule
- [ ] Log project in internal client register

---

## 🚀 Kickoff Meeting Agenda

Use this as the agenda template for the first client meeting:

1. **Introductions** — Team members and roles (5 min)
2. **Project Overview** — Review the brief and objectives together (15 min)
3. **Scope Review** — Walk through in-scope and out-of-scope items (10 min)
4. **Timeline & Milestones** — Confirm key dates (10 min)
5. **Communication Plan** — Establish norms (cadence, channels, escalation) (5 min)
6. **Open Questions** — Address any remaining questions (10 min)
7. **Next Steps** — Confirm immediate action items for both sides (5 min)

---

## ✅ Project Close-Out Checklist

When wrapping up a project:

- [ ] Deliver all final deliverables with client sign-off
- [ ] Collect final payment
- [ ] Archive repository (Settings → Archive this repository)
- [ ] Send close-out survey or request testimonial
- [ ] Document lessons learned in a `/retrospective.md` file
- [ ] Update internal case study / portfolio record
